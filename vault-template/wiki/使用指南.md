# AI 知识库使用指南

> 本指南针对 Obsidian + opencode 环境

---

## 你已经搭好了什么

| 文件/目录 | 用途 |
|---|---|
| `CLAUDE.md` | AI 规则文件，opencode 启动时自动加载 |
| `raw/` | 原始素材，AI 只读不改 |
| `wiki/` | AI 维护的消化笔记 |
| `wiki/index.md` | 全局索引，AI 自动维护 |
| `wiki/log.md` | 操作日志，AI 自动追加 |
| `assets/` | 配图资源 |

## 三个核心操作

### 1. Ingest（录入素材）

看到好文章、好视频笔记、好截图，扔进去：

```
把这个加到 wiki：

[粘贴文章内容 / 贴链接 / 描述素材]
```

AI 会自动：存 raw → 编译成 wiki → 更新索引 → 记日志

### 2. Query（提问）

```
我的 wiki 里关于 XX 有什么？
```

```
根据我的 wiki，总结一下 XX
```

AI 从索引定位相关文章，合成答案并引用。

想让答案存下来：说 **"存下来"** 或 **"归档到 wiki"**。

### 3. Lint（体检）

```
lint wiki
```

AI 检查：索引一致性、链接有效性、孤岛页面、事实矛盾。

> **建议频率：** 每周跑一次。

## 日常工作流

```
1. 打开 Obsidian，在侧边栏找到 opencode 面板
2. 看到好内容 → "ingest 这个：[素材]"
3. 想查什么 → "我的 wiki 里关于 X 有什么？"
4. 每周一次 → "lint wiki" 体检
5. 持续 ingest，知识库复合增长
```

## 注意事项

- **raw/ 里的文件永远不会被 AI 修改**，原始素材是安全的
- vault 里的其他目录 AI 不会动
- 知识库越大越好用，关键是持续 ingest
